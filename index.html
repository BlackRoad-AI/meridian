// Meridian — orchestrate your AI fleet
import { Meridian, Agent } from '@blackroad/meridian';

const fleet = new Meridian({
  project: './my-app',
  memoryBackend: 'sqlite',       // shared context store
  conflictStrategy: 'merge-first', // auto-merge when possible
  agents: [
    Agent.claude({ role: 'architect', scope: 'src/core/**' }),
    Agent.local({  role: 'tester',    scope: 'tests/**' }),
    Agent.gpt({   role: 'reviewer',  scope: '*' }),
    Agent.local({  role: 'docs',      scope: 'docs/**' }),
  ]
});

await fleet.dispatch('Add CRDT sync to the user model');
// Architect designs the approach, tester writes tests,
// reviewer checks for issues, docs updates the guide.
// All coordinated. Zero conflicts.

Orchestration that actually works

Built from real experience running 69 agents across 400+ repos.

🔭

Task Dispatch Engine

Break down complex tasks into subtasks, assign them to the right agents based on capability and scope. Dependency-aware scheduling ensures tasks execute in the correct order.

🔄

Conflict Resolution

When two agents edit the same file, Meridian detects the conflict before it becomes a merge disaster. Automatic three-way merge for compatible changes, human escalation for semantic conflicts.

🧠

Shared Memory Layer

SQLite-backed context store that all agents can read and write. Codex of solutions, TIL broadcasts, journal entries. Every agent builds on what previous agents learned.

📈

Progress Tracking

Real-time dashboard showing which agents are working on what, completion percentage, estimated time remaining. Identify bottlenecks and reassign work dynamically.

🔒

Scope Boundaries

Define which files and directories each agent can modify. Prevent an overeager documentation agent from rewriting your core business logic. Enforce separation of concerns at the agent level.

🔁

Handoff Protocol

When an agent finishes its work or reaches its context limit, Meridian packages the state and hands it off to the next agent. No context lost, no work repeated, seamless continuation.

How a task flows through Meridian

From high-level request to coordinated multi-agent execution.

01

Task Decomposition

Your request is broken into atomic subtasks with dependency relationships. "Add user sync" becomes: design schema, implement CRDT, write tests, update docs, review all.

02

Agent Assignment

Each subtask is matched to the best available agent based on capability, current load, and scope permissions. Architect gets design, tester gets tests, reviewer gets everything.

03

Parallel Execution

Independent tasks run simultaneously in isolated git worktrees. Dependent tasks queue automatically. Memory store keeps all agents aware of each other's progress.

04

Merge + Verify

Completed work is merged through the conflict resolver. Tests run automatically. If anything breaks, the responsible agent is notified and the merge is held until fixed.
